- OOP
  - OOP(Object Oriented Programming)
    - 객체지향프로그래밍
    - 관련된 데이터나 코드를 묶은 객체 컨셉의 프로그램을 하는 패러다임 중 하나
      - 패러다임 : 프로그램을 하는 스타일, 방식
- 객체지향 개념 정리
  - 절차지향적 프로그래밍(Imperative and Procedural Programming)
    - 객체지향 프로그래밍의 반대 개념
    - 정의된 순서대로 절차대로 함수를 하나씩 호출
    - 하나를 수정하기 위해 전체적인 어플리케이션 이해필요
    - 하나를 변경했을 경우 side effect 발생 확률 높음
    - 가독성이 좋지 않음
    - 유지보수성, 확장성 좋지 않음
  - 객체지향적 프로그래밍(Object Oriented Programming)
    - 프로그램을 서로 관련있는 데이터와 함수를 객체로 정의,
      객체들끼리 서로 의사소통하도록 코딩
    - 생산성, 유지보수성, 재사용성, 확장성 높음
    - object
      - 데이터들과 함수들로 구성
        - 데이터 → fields
        - 함수 → methods
    - class
      - object가 어떻게 생겼는지를 묘사(type까지!)
      - template
      - declare once
      - no data in
    - class→**object**
      - class를 이용해서 데이터를 넣은 object 생성
      - instance of a class
      - created many times
      - data in
- 중요한 객체지향 원칙
  - 캡슐화(Encapsulation)
    - 데이터와 함수들 중에서 관련있는 것들을 캡슐화 하는 것
    - 서로 관련있는 데이터나 함수를 한 object 안에 담아두고 외부에서 보여질 필요가 없는 데이터를 잘 숨겨 놓음
  - 추상성(Abstraction)
    - 내부의 복잡한 기능을 모두 이해하지 않고
      외부에서 간단한 interface를 통해 사용할 수 있음
  - 상속성(inheritance)
    - 코드의 재사용성 높임
  - 다형성(polymorphism)
